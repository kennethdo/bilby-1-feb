# Bilby Learnings

## Codebase Patterns
- Next.js App Router with TypeScript
- Tailwind CSS v4 with CSS custom properties for colors (--color-brand, --color-surface, etc.)
- cn() utility from lib/utils.ts for merging Tailwind classes
- MotionSection component wraps sections for scroll-triggered animations
- Typography uses custom Tailwind layer classes (type-display-xl, type-body, etc.)
- Japanese pages use type-jp-* classes, Arabic/GCC pages use type-ar-* classes with RTL direction
- Figma MCP: use get_design_context with fileKey JeBNlyIXgrcyQ1rwyfpebB to read design
- Components follow page-specific naming: about-page.tsx, platform-page.tsx, etc.

## Gotchas
- Express 5.x not relevant — this is Next.js
- globals.css has 40+ typography utility classes — check existing classes before creating new ones
- Motion animations respect prefers-reduced-motion via MotionSection component
- Images are in project root (hero-img.png, etc.) not in public/ directory
- Figma node 3463-1194 is a top-level section "V1.2" containing ALL pages — must drill into specific page frames
- Figma get_design_context output for large nodes saves to file — use python3 to parse XML-like text content
- Motion (Framer Motion) ease strings need `as const` to satisfy TypeScript Variants type
